# Feature Spec: Ingredient Deep-Dive
**Version:** 1.0
**Last Updated:** 2026-04-16
**Status:** Ready for build

---

## What the Feature Does

Ingredient Deep-Dive lets a user tap any product card and see a full breakdown of that product's ingredients. Each ingredient is shown with its name, a safety rating, and a plain-language explanation of what it does and why it is flagged (or not). The goal is to help ingredient-aware users make informed decisions — not just to know that a product is "Caution," but to understand exactly which ingredient caused that rating and why.

The feature surfaces from the product card via an onClick handler. It opens as a full-page detail view (not a modal) and can be navigated back from with a back button. The ingredient list is the centrepiece; the product summary (name, category, overall safety badge) appears at the top as a recap header.

---

## User Flow

1. User sees a product card on Browse, Search, or Shopping List.
2. User taps the card.
3. App navigates to the Product Detail view for that product.
4. User sees a header with the product name, category tag, and overall `SafetyBadge`.
5. Below the header, a scrollable ingredient list renders each ingredient as an `IngredientRow`.
6. Each `IngredientRow` shows: ingredient name, an `IngredientBadge` (clean / caution / avoid), and a short plain-language note explaining the rating.
7. User taps the back button to return to the previous view.

---

## Existing Components Reused

| Component | Where Used in This Feature |
|---|---|
| `SafetyBadge` | Product header — shows the overall product safety score at `size="md"` |
| `CategoryTag` | Product header — displays the product category below the product name |
| `Button` | Back button (`variant="secondary"`) at the top of the detail view |
| `EmptyState` | Shown when no ingredient data is available for the product |

## New Components Needed

### 1. `IngredientBadge`

**Purpose:** A compact color-coded label for a single ingredient's safety rating. Distinct from `SafetyBadge` in that it uses a smaller pill shape and a three-state label set tuned to ingredient-level (rather than product-level) language.

**Props**
| Prop | Type | Required | Notes |
|---|---|---|---|
| `rating` | `'safe' \| 'caution' \| 'avoid'` | yes | Controls color and label text |

**Visual Structure**
```
<span>  rounded-full text-micro font-medium text-neutral-50
        px-sm py-xs

  rating="safe"    bg-success   label: "Safe"
  rating="caution" bg-warning   label: "Caution"
  rating="avoid"   bg-error     label: "Avoid"
```

**Design Tokens Applied**
- Background: `bg-success`, `bg-warning`, `bg-error`
- Text: `text-neutral-50`, `text-micro`, `font-medium`
- Padding: `px-sm py-xs`
- Border radius: `rounded-full`

**Usage Rules**
- Use only on `IngredientRow` — do not use on product cards (use `SafetyBadge` there).
- Label text is always sentence-case.
- Do not repurpose semantic colors for any other use on this page.

---

### 2. `IngredientRow`

**Purpose:** Displays a single ingredient entry — name, safety badge, and a plain-language explanation — as a list item inside the Product Detail view.

**Props**
| Prop | Type | Required | Notes |
|---|---|---|---|
| `name` | string | yes | Ingredient name as it appears on the label |
| `rating` | `'safe' \| 'caution' \| 'avoid'` | yes | Passed to `IngredientBadge` |
| `note` | string | yes | 1–2 sentence plain-language explanation of the rating |

**Visual Structure**
```
<li>  flex flex-col gap-xs
      py-md border-b border-neutral-200
      last: border-none

  <div>  flex items-center justify-between gap-sm
    <span>  text-h4 text-neutral-900      — ingredient name
    <IngredientBadge rating={rating} />

  <p>  text-body text-neutral-600         — plain-language note
```

**Design Tokens Applied**
- Padding: `py-md`
- Border: `border-b border-neutral-200`
- Name: `text-h4 text-neutral-900`
- Note: `text-body text-neutral-600`
- Internal gap: `gap-xs` between name row and note, `gap-sm` between name and badge

**Usage Rules**
- Always render inside a `<ul>` with `list-none`.
- The bottom border separates rows; the last row has no border (`last:border-none`).
- Never truncate the `note` — ingredient explanations must be fully readable.

---

### 3. `ProductDetailPage`

**Purpose:** Full-page view that shows a product's complete ingredient breakdown. Accessed by clicking any `ProductCard`.

**Location:** `src/features/product/ProductDetailPage.jsx`

**Props / State**
Receives a `product` object (id, name, category, safety_score, description) and an `ingredients` array fetched from Supabase or generated by the Claude API on load.

**Visual Structure**
```
<main>  min-h-screen bg-neutral-50 px-lg py-xl

  <div>  max-w-2xl mx-auto flex flex-col gap-xl

    — Back button
    <Button variant="secondary" onClick={onBack}>← Back</Button>

    — Product header
    <header>  flex flex-col gap-sm
      <div>  flex items-start justify-between gap-sm
        <h1>  text-h1 font-serif text-neutral-900     — product name
        <SafetyBadge score={safety_score} size="md" />
      <CategoryTag label={category} />
      <p>  text-body text-neutral-600                 — product description

    — Ingredient list
    <section>
      <h2>  text-h2 font-serif text-neutral-900 mb-md  — "Ingredients"
      <ul>  list-none
        {ingredients.map(i => <IngredientRow ... />)}
```

**States**
| State | Treatment |
|---|---|
| Loading | Centered `text-body text-neutral-600` "Loading ingredients…" |
| Loaded | Full ingredient list rendered |
| Empty / no data | `EmptyState` with heading "No ingredient data" and body "We don't have ingredient information for this product yet." |
| Error | `text-body text-error` error message centered on the page |

**Design Tokens Applied**
- Page background: `bg-neutral-50`
- Page padding: `px-lg py-xl`
- Content max-width: `max-w-2xl mx-auto`
- Section gap: `gap-xl`
- Heading: `text-h2 font-serif text-neutral-900`

## Data Shape

Each ingredient entry returned for the feature:

```js
{
  name: string,          // e.g. "Sodium Lauryl Sulfate"
  rating: 'safe' | 'caution' | 'avoid',
  note: string           // e.g. "A surfactant that can strip natural oils and irritate sensitive skin."
}
```

Ingredient data is sourced from the EWG Skin Deep API or generated by Claude when EWG data is unavailable. All API calls go through `src/lib/api/` — never inline in components.

---

## Routing

`ProductDetailPage` is a new in-app route. App.jsx passes the selected product via state (no URL param needed in V1). `ProductCard`'s existing `onClick` prop is used as the entry point. The calling page passes an `onBack` callback to return to the previous view.

`ProductCard` already accepts `onClick` — no changes to that component are needed.

## Acceptance Criteria

- [ ] Clicking any `ProductCard` on Browse, Search, or Shopping List navigates to the Product Detail view for that product.
- [ ] The detail view header shows the product name (`text-h1`), overall `SafetyBadge` at `size="md"`, `CategoryTag`, and description.
- [ ] Each ingredient renders as an `IngredientRow` with name, `IngredientBadge`, and a plain-language note.
- [ ] `IngredientBadge` uses `bg-success` for safe, `bg-warning` for caution, `bg-error` for avoid — no other colors.
- [ ] The ingredient list is separated by `border-neutral-200` dividers; the last row has no bottom border.
- [ ] A loading state is shown while ingredient data is being fetched.
- [ ] An `EmptyState` is shown when no ingredient data is available.
- [ ] The Back button returns the user to the view they came from (Browse, Search, or Shopping List) without a full page reload.
- [ ] No hex values, pixel sizes, or spacing values are hardcoded — all tokens come from `tailwind.config.js`.
- [ ] `IngredientBadge` and `IngredientRow` are added to `component-spec.md` before build begins.
- [ ] All API calls for ingredient data go through `src/lib/api/` — not inline in any component.
